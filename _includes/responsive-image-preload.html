{%- comment -%} responsive-image-preload.html — Emits tag(s) for a hero/LCP image, using the same variant/srcset resolution as responsive-image.html. A plain href-only preload would tell the browser to fetch the raw original while the actual (via responsive-image.html) requests a differently-sized srcset candidate — the preload is then never used, wasting bandwidth and triggering a "resource was preloaded but not used" browser warning. Passing the matching imagesrcset/imagesizes lets the browser's preload candidate selection agree with the tag's. Parameters (mirrors responsive-image.html): src — raw image path (site-root relative or absolute). sizes — sizes attribute the corresponding will use. fallback — fallback image path if no variant/original exists. fetchpriority — defaults to 'high' (this include exists for LCP images). all_static_paths — precomputed '||'-joined site.static_files list, optional. {%- endcomment -%} {%-
